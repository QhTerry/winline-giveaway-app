# Winline Giveaway — Telegram Mini App

Role-aware SPA для розыгрышей в фирменном чёрно-оранжевом стиле Winline
(прямой шрифт, кнопки-пилюли, графитовые карточки). Статический фронтенд —
деплоится на Vercel без сборки. **Сервис стартует пустым** — без демо-данных.

## Что уже работает

**Навигация:** нижний таб-бар, зависит от роли. Участник видит «Мои / Профиль»,
у админа добавляется «Админка». Общей ленты-каталога нет: розыгрыш открывается
**по deep-link из поста в канале** (`?ga=ID` / Telegram `start_param`) или из
«Моих». Роль для демо переключается в Профиле (в бою выдаётся по TG-юзеру).

**Участник:**
- **Онбординг** при первом входе (шапка и таб-бар скрыты).
- **Карточка розыгрыша** — приз, условия со статусами, кнопки «Пост в канале» и
  «Правила», участие. Без какой-либо статистики розыгрыша.
- **Участие** — авторизация Winline + пошаговая проверка условий (заглушка).
- **Мои** — только розыгрыши, где участвует пользователь: участия + выигрыши.
- **Выигрыш** — экран получения приза.
- **Итоги (read-only)** — список победителей + проверочный хеш, без WL ID и
  без действий админа.
- **Правила / FAQ** — из карточки и из профиля.
- **Профиль** — привязка/отвязка Winline, 18+, переключатель роли (демо).

**Админ:**
- **Дашборд** — свои розыгрыши: идут / черновики / завершённые.
- **Модульный конструктор** — оформление + добавляемые блоки условий (подписка /
  промокод / ставка-депозит; обязательных нет), места, кросспост-каналы, дата.
  Живой предпросмотр. Публикация и черновики.
- **Управление** — пост в канале, предпросмотр «как участник», пауза/возобновление,
  досрочное завершение, дублирование, редактирование, кросспостинг.
- **Итоги** — победители (с WL/TG ID) + проверочный хеш (provably-fair) + рерол.
- **Аналитика** — KPI, таблица условий, выгрузка в Google Sheets (только админ).
- **Команда** (супер-админ) — выдача/отзыв доступа по @username.

Интеграция с **Telegram WebApp SDK**: тема, кнопка «Назад», haptic, разворот,
`openTelegramLink` для перехода к посту.

> Демо-режим: данные в localStorage, авторизация и проверка условий — заглушки.
> Реальные интеграции (Winline API, проверка подписок, Google Sheets, постинг в
> канал, планировщик итогов) помечены в `app.v5.js` комментариями `TODO:BACKEND`.

## Деплой на Vercel

**Вариант A — через GitHub (рекомендую):**
1. Залей папку `winline-giveaway-app` в репозиторий GitHub.
2. На vercel.com → **Add New → Project** → импортируй репозиторий.
3. Framework Preset: **Other**. Build Command — пусто. Output Directory — пусто
   (корень). Жми **Deploy**.

**Вариант B — через CLI:**
```bash
npm i -g vercel
cd winline-giveaway-app
vercel        # preview
vercel --prod # продакшн
```

Получишь URL вида `https://winline-giveaway.vercel.app`.

## Подключение как Telegram Mini App

1. В Telegram открой **@BotFather** → `/newapp` (или `/mybots` → выбрать бота →
   **Bot Settings → Configure Mini App**).
2. Укажи URL твоего деплоя на Vercel.
3. Открой мини-апп через меню бота или кнопку — он развернётся на весь экран
   и подхватит тему Telegram.
