# Winline Giveaway — продуктовый план v1

Документ фиксирует, как устроен мини-апп: роли, навигация, экраны, конструктор
розыгрыша, механика честности, призы, бэкенд и дизайн. По нему сверяемся и режем
работу на этапы.

---

## 1. Роли и доступ

Три роли в одном приложении, интерфейс адаптируется под роль (role-aware).

| Роль | Кто это | Что может |
|------|---------|-----------|
| **Участник** | любой пользователь Telegram | смотреть ленту, участвовать, видеть свои участия/выигрыши, привязывать Winline |
| **Админ** | конкретные TG-юзеры, которым выдали доступ | всё, что участник + создание и управление своими розыгрышами, аналитика |
| **Супер-админ** | мы (центр) | всё + выдача/отзыв доступа админам, видит все розыгрыши |

**Выдача доступа** — по TG (username или ID). Супер-админ добавляет юзера в список
админов прямо в аппе (экран «Команда»). Админ видит только свои розыгрыши;
супер-админ — все.

Разделение визуально: у участника в нижнем баре 3 вкладки, у админа появляется
4-я — **«Админка»**. Один апп, одна точка входа, разный набор вкладок.

---

## 2. Навигация (нижний таб-бар, role-aware)

**Участник:**
- 🎁 **Лента** — каталог активных розыгрышей с таймерами
- 🎫 **Мои** — мои участия (статусы условий) + история выигрышей
- 👤 **Профиль** — привязка Winline ID, статус, выход

**Админ — добавляется вкладка:**
- 🛠 **Админка** — мои розыгрыши, создание, аналитика, (для супер-админа — команда и все розыгрыши)

Вход в конкретный розыгрыш — двумя путями:
1. **Кнопка под постом в канале** → deep link открывает мини-апп сразу на этом розыгрыше.
2. **Лента внутри аппа** → карточка розыгрыша.

---

## 3. Карта экранов

### Участник
1. **Лента** — карточки: обложка, приз, таймер до конца, краткие условия, кнопка участия.
2. **Карточка розыгрыша** — полное описание, список условий с живыми статусами
   (🟢 выполнено / 🔴 нет / → действие), кнопка участия.
3. **Участие** — если Winline не привязан → авторизация; проверка условий; подтверждение.
4. **Мои участия** — список с прогрессом условий, «доделать» подсказки.
5. **Мои выигрыши / история** — архив, статус получения приза, инструкции.
6. **Профиль** — Winline ID (маскированный), статус привязки, смена/выход, 18+/верификация.

### Админ
7. **Админка (дашборд)** — мои розыгрыши (идёт/завершён/черновик), кнопка «Создать».
8. **Конструктор розыгрыша** — оформление + модульные условия + публикация (см. §4).
9. **Управление розыгрышем** — участники в реал-тайм, пауза/досрочное завершение,
   продление, редактирование, дублирование, кросспостинг.
10. **Подведение итогов** — список победителей, рерол, публикация, проверочный хеш.
11. **Аналитика** — KPI, таблица участников (статусы условий), выгрузка в Google Sheets.
12. **Команда** (только супер-админ) — список админов, добавить по @username, отозвать.

## 4. Конструктор розыгрыша (модульный)

Ключевое отличие от старого ТЗ: **нет обязательных условий по умолчанию**. Админ
собирает розыгрыш из блоков, любое условие добавляется по желанию, можно несколько
однотипных.

**Оформление:** обложка (загрузка или пресет-шаблон), текст, текст кнопки.

**Условия (добавляются кнопкой «+ Добавить условие»):**
- **Подписка на канал** — один или несколько каналов (бот должен быть админом).
- **Регистрация по промокоду** — промокод актива + динамическая UTM (utm_source/utm_promo)
  под кросс-канал.
- **Ставка / депозит** — мин. сумма X ₽, категория (все события / киберспорт /
  конкретное событие по ссылке), за период розыгрыша.

*(Рефералы — отложены, заложим расширяемость.)*

**Параметры розыгрыша:** количество мест (победителей), канал(ы) публикации
(кросспостинг), дата/время итогов или «сейчас».

**Состояния:** черновик → запланирован → идёт → завершён.

## 5. Механика и честность

- **Определение победителей:** случайный выбор N из участников, выполнивших все
  заданные условия, по таймеру (как в классических TG-розыгрышах).
- **Прозрачность / проверяемость:** до старта розыгрыша фиксируем и публикуем
  **хеш сида**; после — публикуем сид и список, чтобы любой мог пересчитать
  результат (provably-fair, как в популярных ботах-розыгрышниках).
- **Рерол:** админ может перевыбрать победителей (с логированием).

**Анти-фрод:**
- **1 Winline ID = 1 Telegram** — жёсткая привязка, один логин нельзя привязать к
  другому TG (проверка по всей базе).
- **Чёрный список** — интеграция с вашей Google-таблицей ЧС (ботоводы/мультиаккаунты):
  такие участники исключаются из выборки.
- Задел: гео/возраст 18+ и верификация через Winline перед выдачей приза.

## 6. Призы и выдача

Призы разнотипные, у каждого свой сценарий выдачи:

| Тип приза | Выдача |
|-----------|--------|
| Фрибет / бонус / деньги на баланс | потенциально авто-начисление через Winline API |
| Девайсы / мерч | сбор адреса/контактов в аппе → ручная отправка |
| Промокоды / цифровое | выдача в аппе/личке автоматически |
| Билеты / мероприятия | ручная выдача, инструкция в личке |

**Уведомление победителей:** пост в канале со списком + личка в боте каждому +
получение/подтверждение приза в мини-аппе (с дедлайном; задел на авто-замену, если не забрал).

## 7. Бэкенд и интеграции (на Vercel)

Сейчас фронт работает на демо-данных (localStorage). Для боевой версии:

- **Хранилище:** Postgres (Supabase/Neon) — розыгрыши, участники, связки
  Winline ID ↔ Telegram ID, ЧС, логи розыгрыша.
- **API:** serverless-функции Vercel (`/api/*`).
- **Telegram-бот:** webhook — приём /start, deep links, постинг карточек и итогов,
  личка победителям, проверка подписок (бот-админ каналов).
- **Winline API:** авторизация/привязка, проверка промокода, ставки/депозита,
  (по возможности) начисление призов. *Нужны доступ и документация.*
- **Google Sheets API:** выгрузка участников + чтение таблицы ЧС.
- **Планировщик:** cron (Vercel Cron) для авто-подведения итогов по времени.
- **Безопасность:** валидация Telegram `initData` (подпись), проверка ролей на бэке.

## 8. Дизайн-система (чёрно-оранжевый Winline)

Полное фирменное оформление в стиле Winline: графитовый/чёрный фон, оранжевые
акценты, белый текст, спортивная энергия. На этапе вёрстки снимем точные токены
с сайта winline.ru (цвета, типографика, скругления, тени кнопок) и приведём к ним.

**Текущая палитра (база, уточним по брендбуку):**
- Оранжевый `#F58220` (акцент/кнопки), свет `#FFA24D`, тёмный `#E26F12`
- Графит фон `#121212`, поверхности `#1B1B1B` / `#262626`, границы `#333`
- Текст белый `#FFFFFF`, приглушённый `#9AA0A6`
- Статусы: зелёный `#34C759`, красный `#FF453A`

**Принципы:** крупные читаемые заголовки, карточки со скруглением 14–18px,
оранжевые CTA с мягким свечением, единый набор линейных иконок (уже есть),
микро-анимации (таймеры, появление, розыгрыш). Логотип — воссозданный SVG.

## 9. Роадмап

**Этап 0 — готово:** кликабельный прототип + первая рабочая демо-версия (визард,
мои розыгрыши, итоги, аналитика, превью участника), деплой на Vercel.

**Этап 1 — UX/дизайн (frontend, без бэка):**
1. Снять брендинг с winline.ru, обновить дизайн-токены.
2. Ввести role-aware нижний таб-бар (Лента / Мои / Профиль / Админка).
3. Сделать ленту розыгрышей + карточку + экран участника как полноценные экраны.
4. Переделать визард в модульный конструктор условий.
5. Экраны «Мои участия», «Выигрыши», «Профиль», «Команда».

**Этап 2 — бэкенд и интеграции:**
6. БД + API + Telegram webhook (deep links, постинг, проверка подписок).
7. Авторизация/привязка Winline + проверка условий (промокод, ставка).
8. Планировщик итогов, provably-fair, личка победителям.
9. Google Sheets: выгрузка + чтение ЧС.

**Этап 3 — призы и полировка:**
10. Сценарии выдачи призов (баланс/мерч/промо/билеты), получение в аппе.
11. Авто-замена невостребованных, верификация 18+, анти-фрод правила.
12. Финальная анимация/полировка, тесты.
